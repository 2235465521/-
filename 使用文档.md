# 标准 PDF 下载 — 使用文档

> 版本：3.0.0（Web 界面 v4.1）  
> 适用：日常检索、高级筛选、批量下载

---

## 1. 系统简介

**标准PDF下载** 是一套本地/内网标准 PDF 检索与下载工具，通过浏览器访问。主要能力：

| 模块 | 说明 |
|------|------|
| **标准检索** | 按标准号、名称检索国标/行标/企标等，支持高级筛选 |
| **同类产品** | 输入产品名（如「铅笔」），自动扩展同类关键词检索 |
| **团标征求意见稿** | 按协会名或标准名称检索团标 PDF |
| **批量下载** | 上传 Excel/CSV 清单，自动匹配并打包 ZIP |

数据来源：MySQL 标准库（或本地 `data/standards.db`）+ 磁盘 PDF 目录。

## 2. 启动与停止

### 2.1 启动（推荐）

双击以下任一文件（效果相同）：

- **`启动ZKBZ.bat`**（推荐，文件名保留，界面名称为「标准PDF下载」）
- **`启动.bat`**
- **`start_zkbz.bat`**

启动成功后：

- 黑窗口保持打开（**关闭即停止服务**）
- 浏览器自动打开：**http://127.0.0.1:5000/**

### 2.2 首次运行

脚本会自动检查 Python 依赖；若缺包会安装 `requirements.txt`。

若提示缺少索引文件，**不会**在启动时自动构建（避免卡几小时），需按 [第 6 节](#6-索引构建首次或重建) 手动构建。

### 2.3 停止

直接关闭启动时的黑色命令行窗口即可。

## 3. 界面说明

### 3.1 左侧导航

- **标准检索**：主功能，编号/名称 + 高级筛选
- **同类产品**：产品名扩展检索，侧栏可点产品组试搜
- **团标征求意见稿**：团标 PDF 目录检索
- **批量下载**：Excel 清单批量打包

### 3.2 顶部搜索框

- 输入标准编号或名称关键词，点 **检索**
- 下方 **最近检索** 可一键复用历史关键词（仅标准检索模式）

### 3.3 结果表格

- 点击行可展开，查看 PDF 文件列表
- 勾选左侧复选框 → 点 **多项下载** 打包当前选中项
- 结果上方蓝色条显示当前生效的高级筛选条件

## 4. 标准检索

### 4.1 基本检索

示例：

- 标准号：`GB/T 16472-2024`
- 名称关键词：`煤矿`、`健康信息学`

### 4.2 高级筛选

点 **高级选项** 展开面板，常用字段：

| 字段 | 说明 |
|------|------|
| 状态 | 现行 / 即将实施 / 废止 |
| 标准类型 | 国标、行标等 |
| 省 / 市 / 县 | 按**全部起草单位**通讯地址归类 |
| 产品/品种 | 可手输或下拉联想，命中词库会扩展关键词 |
| 公司/起草单位 | 输入 2 字以上自动联想，如输入「福建」列出相关单位 |
| 起草顺位 | 须配合「公司/起草单位」使用，见下表 |
| 年份起 / 年份止 | 可按发布年份范围筛选 |

**起草顺位选项：**

| 选项 | 含义 |
|------|------|
| 不限 | 该单位出现在任一顺位即可 |
| 第 1 / 2 / 3 起草单位 | 仅匹配该单位排在对应位次的标准 |
| 大于三 | 该单位排在第 4 位及以后 |

示例：查「福建理工大学」作为**第一起草单位**的标准：

1. 公司/起草单位：`福建理工大学`
2. 起草顺位：**第 1 起草单位**
3. 点 **应用筛选**

> 注意：顺位筛选必须同时填写单位名；只选顺位不填单位不会生效。

### 4.3 筛选记录

应用筛选后，面板下方会出现 **筛选记录（点击复用）**。点击某条即可恢复当时的条件并重新检索，适合重复下载场景。

### 4.4 地区批量下载

1. 高级筛选中**必须先选择省份**
2. 可叠加市、县、产品、单位、年份等条件
3. 点 **地区批量下载** → 确认数量 → 打包 ZIP  
4. 单次最多 **5000** 条

---

## 5. 同类产品检索

1. 左侧切换到 **同类产品**
2. 输入产品名，如：`牙膏`、`铅笔`、`阀门`
3. 系统按产品词库扩展同类关键词后检索
4. 切换回「标准检索」时，搜索框内容**互不带入**（各模式独立记忆）

---

## 6. 团标征求意见稿

1. 左侧切换到 **团标征求意见稿**
2. 输入协会名或标准名称关键词
3. 结果中可直接下载 PDF

需已构建团标索引（见下节）。

---

## 7. 批量下载

1. 左侧切换到 **批量下载**
2. **下载模板** → 填写标准号（可选名称）
3. **上传** `.xlsx` / `.csv`（单次最多 **400** 条）
4. 预览匹配结果 → **打包下载 ZIP**

勾选 **扫描磁盘** 时，会在 E 盘等配置的 PDF 根目录下补充查找文件。

---

## 8. 索引构建（首次或重建）

启动脚本**不会**自动构建大索引。若功能不可用，在项目目录打开命令行执行：

```bat
python scripts\build_index.py
```

构建标准库 SQLite 索引（`data/standards.db`）。

```bat
python scripts\build_unit_index.py
```

构建起草单位地址索引（`data/units.db`），用于省/市/县、单位名、起草顺位筛选。**耗时较长（约数十分钟）**，只需构建一次。

```bat
python scripts\build_tuangbiao_index.py
```

构建团标索引（`data/tuangbiao.db`）。

## 9. 配置说明

项目根目录 `.env` 可配置（无则使用默认）：

| 变量 | 说明 | 默认 |
|------|------|------|
| `PORT` | 服务端口 | `5000` |
| `OPEN_BROWSER` | 启动后是否打开浏览器 | `true` |
| `PDF_ROOT` | 标准 PDF 根目录 | 自动检测 E:/Z: 盘 |
| `MYSQL_HOST` / `MYSQL_PORT` / `MYSQL_USER` / `MYSQL_PASSWORD` / `MYSQL_DATABASE` | MySQL 连接 | 本地 mydate |
| `SQL_DUMP_DIR` | SQL 导出目录（建索引用） | 见 paths.py |

修改 `.env` 后需**重启**启动脚本。

## 10. 常见问题

### 启动后浏览器打不开

- 确认黑窗口无报错，访问 http://127.0.0.1:5000/
- 若端口被占用，启动脚本会先尝试结束旧进程

### 检索一直转圈 / 很慢

- 按 **Ctrl+F5** 强制刷新页面
- 重启 `启动ZKBZ.bat`
- 单位联想输入至少 **2 个字** 才会请求

### 选了「第 1 起草单位」仍出现非第一单位的标准

- 确认已填写 **公司/起草单位** 并点了 **应用筛选**
- 看结果上方蓝色筛选条是否含「仅第 1 位」
- 部分标准该单位排第一但**无 PDF 记录**，默认只显示有 PDF 的条目

### 按省/市/单位筛选无结果

- 确认已运行 `python scripts\build_unit_index.py`
- 检查 `data\units.db` 是否存在

### 页面样式或功能不对

- **Ctrl+F5** 强刷清除缓存
- 确认启动的是最新版 `启动ZKBZ.bat`
